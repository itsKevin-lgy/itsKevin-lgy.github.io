关于此次部署静态网页博客的总结与反思

发表于 2023-03-30

前言：读了很多年书，但是从来没有想过要有一个属于自己的博客，之前做记录的方式就是用OneNote，例如阅读论文，随手记下。但是这次开源课程作业给了一个部署静态博客的机会，以后也尽量去更新自己的研究进展，也不荒废自己这几天的辛苦搭建…

博客主题及其选取原因

本次作业采用了Hexo+Github部署，使用的主题是NexT主题。Hexo是一个基于Node.js的静态博客框架，它可以快速生成静态博客页面并且易于部署。而NexT是一款流行的Hexo主题，它具有简洁、美观、易用等特点，因此备受Hexo用户的喜爱。NexT主题支持多种布局和自定义选项，使得用户可以根据自己的需求来定制博客的外观和功能。我选择该主题的原因是因为NexT主题具有现代、简洁的设计风格，使得博客页面看起来非常美观和舒适。同时，NexT主题提供了丰富的自定义选项和插件支持，可以轻松地修改博客的外观和功能，满足个性化需求。

博客页面布局及其设计思路

博客包括主页面、关于我、标签、分类、归档五项。在关于我中介绍了自己的详细信息。目前共计发布了两篇日志，第一篇是阅读论文“Weakly Supervised Anomaly Detection:A Survey”后的随手笔记，标签是“paper”；第二篇则是该篇总结和感悟，标签是“blog”。

博客功能实现及其技术选择

博客是基于Hexo部署的，使用 Hexo部署静态博客的基本流程如下：
1.安装 Node.js：在你的电脑上安装 Node.js，可以通过官网下载对应系统的安装包并安装。
2.安装 Hexo：在命令行中使用 npm 安装 Hexo，输入以下命令：
npm install -g hexo-cli
3.创建Hexo博客：使用 Hexo 命令创建一个新的博客，输入以下命令：
hexo init myblog
cd myblog
npm install
4.配置 Hexo 博客：打开 myblog 文件夹下的 _config.yml 文件，根据需要修改网站标题、描述、作者等信息。
5.创建文章：使用 Hexo 命令创建一篇新文章，输入以下命令：
hexo new "Hello World"
6.写作文章：在 source/_posts 目录下找到新创建的文章，并编辑 Markdown 格式的文章。
7.预览博客：使用 Hexo 命令在本地预览博客，输入以下命令：
hexo server
然后在浏览器中输入 http://localhost:4000 地址即可预览博客。
8.部署博客：使用 Hexo 命令将博客部署到远程服务器，输入以下命令：
hexo deploy
部署完成后，在浏览器中输入你的博客地址即可访问。

博客样式设计及其美学考量

我选择的样式是NexT主题中的Gemini，因为它吸引我的最大特点是风格简洁，这样可以让读者专注于文章内容本身，而不会被花哨的设计分心。此外，Gemini样式采用了一些优秀的排版技巧，如宽度适中、行间距合适、字体清晰等等，可以提供更好的阅读体验。
博客的背景是最喜欢的球星科比，为了不影响阅读体验，设置了透明度。
同时，设置了阅读全文选项，在主页仅展示日志的摘要部分。

博客制作过程中遇到的问题及其解决方法

在设置阅读全文选项时，网上的教程大部分都是修改NexT主题配置文件中的auto_excerpt，但是我发现，找不到auto_excerpt…
是因为新版NexT主题已经删除了这个选项
可以先安装插件，再进行相关配置就可以启用首页预览。
具体步骤如下（参考：https://www.zhihu.com/question/394167076）：
1.在根目录下执行：
npm install hexo-excerpt --save
2.在站点配置文件里添加：
excerpt:
depth: 5
excerpt_excludes: []
more_excludes: []
hideWholePostExcerpts: true
3.主题配置文件设置：
excerpt_description: true
…
read_more_btn: true
（by me:这里应该默认就是true）
